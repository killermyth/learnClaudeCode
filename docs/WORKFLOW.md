# Claude Code 工作流程

## 请求处理流程

```
用户请求
    ↓
请求解析和理解
    ↓
任务分解
    ↓
执行规划
    ↓
工具调用
    ↓
结果验证
    ↓
反馈用户
```

## 关键决策点

### 1. 工具选择
- 分析任务类型
- 评估可用工具
- 选择最优方案

### 2. 执行顺序
- 识别任务依赖
- 制定执行计划
- 支持并行处理

### 3. 错误恢复
- 监控执行状态
- 处理异常情况
- 提供补救方案

## 交互模式

### 主动式
- 主动收集必要信息
- 做出合理假设
- 持续推进任务

### 反馈式
- 及时提供进展信息
- 清晰的结果报告
- 必要时请求确认

## 代码修改流程

1. **理解现状**
   - 读取相关文件
   - 分析代码结构
   - 确定修改位置

2. **制定方案**
   - 设计修改策略
   - 评估影响范围
   - 规划实施步骤

3. **执行修改**
   - 使用合适工具
   - 保证修改精确
   - 避免不必要的改动

4. **验证结果**
   - 检查修改是否完成
   - 验证代码正确性
   - 测试功能完整性

## 最佳实践

1. **信息收集** - 充分理解需求前再行动
2. **工具优化** - 批量操作优于顺序操作
3. **增量开发** - 小步迭代而非一步到位
4. **质量保证** - 验证每个修改的正确性
5. **清晰沟通** - 提供进展和结果反馈
